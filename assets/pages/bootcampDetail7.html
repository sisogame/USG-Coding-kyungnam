7.RESTFul, SAP-RFC

정의

  • · 두 컴퓨터 시스템이 인터넷을 통해 정보를 안전하게 교환하기 위한 인터페이스
  • · HTTP를 기반으로 하며, 자원을 URI로 표현하고, HTTP 메서드를 사용하여 자원
  • · 웹 서비스의 핵심 개념이며, 다양한 웹 서비스
  • · RFC(Remote Function Call)는 같은 시스템(SAP SW 및 ABAP 프로그래밍)이나 다른 시스템(C++, VB, JAVA 등의 프로그래밍)의 호출자로부터 Function Module을 불러와 실행하는 Function tools

편성 배경

  • · 시스템 간 통합과 데이터 교류를 위한 실무기솔로 RESTful API와 RFC의 역할 확대
  • · 입문자에게 필요한 RESTful 및 SAP-RFC 기초 지식 습득과 실습 중심 교육의 필요성 인식
  • · 효과적인 API 개발과 최적화 전략에 대한 교육으로 개발자의 문제해결능력 강화
  • · 현대 SW아키텍처에 필수적인 RESTful API 보완과 관련된 예제와 실습 제공 필요

학습목표

  • 목표
    RESTful, SAP-RFC

  • · RESTful 아키텍처 기본 원리 및 API 디자인: 웹서비스의 핵심인 RESTful아키텍처의 이해와 RESTful API의 구성요소와 디자인 원칙에 대한 체계적인 교육 제공

    · RESTful API 개발과 최적화 실습: HTTP 메서드 활용, URL설계, 상태코드 관리와 RESTful API의 성능, 확장성, 유지보수성최적화 기법에 대한 실용적인 실습

    · SAP-RFC와 시스템통합 기술: 다양한 시스템 간 통신을 가능하게 하는 SAP-RFC의 기능 이해 및 실제 시스템에 적용하는 통합 기술 학습
